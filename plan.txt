表8：第二周任务分解与验收标志
第二周技术要点
关键帧筛选使用拉普拉斯算子计算清晰度，设置阈值排除模糊帧，在PSRAM中维护循环缓冲区（30帧约3MB）

第三周技术要点

四、关键模块技术方案
4.1 语音唤醒与ASR/TTS对接方案
语音交互是整个系统的用户入口，需要保证稳定、低延迟。推荐使用乐鑫赛道提供的百度飞桨星河社区大模型API。
唤醒词检测：使用ESP-SR的WakeNet模型，支持"小智小智"或自定义唤醒词，端侧离线运行
ASR识别：唤醒后录音5秒，通过HTTP POST发送到百度飞桨ASR API，返回文本结果
VLM问答：将用户问题+当前关键帧（JPEG Base64编码）发送到云端VLM，返回文本答案
TTS播报：将VLM答案文本发送到TTS API，获取PCM音频流，通过I2S实时播放
语音交互完整链路时延预估：唤醒检测(200ms)+录音(5s)+ASR(500ms)+VLM(2-3s)+TTS(500ms)+播放(2s)=约10-12s。第一次交互约10s，后续可优化至5-7s。
4.2 关键帧筛选算法
关键帧筛选是本项目的核心创新点，目标是在端侧排除模糊、无信息量的帧，减少云端调用次数。设计简化版关键帧评分函数：
Score = alpha * Laplacian(img) + beta * MotionDiff(prev, curr)

if Score > THRESHOLD:
    save_to_keyframe_buffer(img, timestamp, Score)

Laplacian使用OpenCV实现，计算图像清晰度；
MotionDiff使用帧差分析，检测画面变化。
缓冲区设计：在PSRAM中维护环形缓冲区，容量30帧，每帧约100KB（640x480 JPEG），总计约3MB
用户唤醒时从缓冲区选取评分最高的一帧作为"当前关键帧"发送给VLM
事件触发时保存事件前中后三帧，组成"事件关键帧组"
4.3 端侧人体检测（ESP-WHO）
人体检测采用ESP-WHO框架在端侧本地运行，不依赖云端，响应速度快。
模型选择：使用ESP-WHO预置的轻量人体检测模型，适配ESP32-P4的PIE AI加速指令
检测流程：摄像头每5帧取1帧送入ESP-WHO推理，输出人体存在置信度
与PIR融合：PIR中断触发后立即启动ESP-WHO检测进行交叉验证，降低误触发率
资源占用：模型约占用200KB PSRAM，推理耗时约50-100ms/帧
安全降级：如果ESP-WHO运行不稳定，可回退到纯PIR检测模式
4.4 云端VLM对接方案
云端VLM对接采用标准HTTP REST API方式，推荐使用百度飞桨星河社区提供的大模型API。请求/响应格式设计如下：
POST /v1/chat/completions
Content-Type: application/json
Authorization: Bearer {token}

{
  "model": "ernie-lite-v1",
  "messages": [
    {
      "role": "user",
      "content": [
        {"type": "text", "text": "用户问题"},
        {"type": "image_url", "image_url": {"url": "data:image/jpeg;base64,..."}}
      ]
    }
  ]
}
图片压缩：将JPEG图片压缩至100-150KB，Base64编码后约200KB，控制单次请求体积在500KB以内
缓存机制：对于相同问题+相似画面的请求，端侧维护简单本地缓存（问题文本哈希+答案），避免重复调用
故障处理：云端请求失败时，端侧播报"网络异常，请稍后再试"，屏幕显示错误状态图标
4.5 传感器融合与规则引擎
规则引擎采用简化的状态机实现，支持两种预设模板：
规则A（画面变化提醒）：当画面帧差超过阈值时，保存关键帧并语音播报提醒
规则B（人走灯灭）：当PIR检测到无人且光照值仍高于阈值时，语音询问是否关灯，确认后LED熄灭
规则引擎状态机设计：
IDLE -> 用户配置规则 -> MONITORING
MONITORING -> 传感器/视觉触发 -> TRIGGERED
TRIGGERED -> 云端确认完成 -> CONFIRMING
CONFIRMING -> 用户确认或超时 -> ACTION
ACTION -> LED灯带动作 -> IDLE
4.6 物联网执行层方案（LED灯带）
物联网执行层通过LED灯带模拟灯光开关状态，安全策略如下：
LED灯带属于低风险动作，触发后直接执行，无需用户确认
灯光状态表示：PWM占空比80%+白色=灯亮（模拟开灯），PWM占空比0%=灯灭（模拟关灯）
所有动作执行前检查白名单，未在白名单中的动作拒绝执行并记录日志
LED灯带通过GPIO PWM控制，无需额外驱动电路，直接接开发板GPIO即可

5.2 备选方案汇总
如果某个模块遇到不可解决的问题，可以采用以下备选方案确保MVP仍然可以完整演示：
语音交互不稳定：可以改为触屏按钮触发拍照和显示答案，保留核心VLM问答能力
云端连接不稳定：可以将常见问题的答案缓存在本地SPIFFS，网络不通时返回本地答案
ESP-WHO人体检测失败：回退到纯PIR检测模式，PIR输出高低电平即可判断有人/无人
屏幕显示异常：可以改为通过串口打印日志，或使用手机Web端显示状态
