二、阶段一：云端预训练与微调
2.1 基础模型选择：OSNet-IBN-x0.25
选择理由： - OSNet（Omni-Scale Network）的核心设计是多尺度特征融合，通过动态聚合不同感受野的特征来捕捉行人图像中的细节和全局信息。这种设计对上半身场景尤为重要，因为上半身身份线索（如肩宽、头型、 torso 轮廓）分布在不同尺度。 - IBN（Instance-Batch Normalization）模块是跨域/跨外观鲁棒性的关键：Instance Normalization 对 appearance 变化（颜色、纹理）不敏感，Batch Normalization 保留身份判别信息。这种组合天然适合换装场景。 - x0.25 是宽度乘子为 0.25 的精简版，参数量约 200K-500K，是 OSNet 系列最轻量的变体。
模型结构理解：
输入：256×128×3（标准）或 128×64×3（你的上半身适配）      ↓Stem：Conv 3×3 → BN → ReLU → MaxPool      ↓OSNet Block × 4（核心）：  - 每个 Block 包含多个分支，分别用 1×1、3×3、5×5 等卷积提取不同尺度  - 动态聚合：通过学习到的权重融合多尺度特征  - IBN 层嵌入：前半通道用 IN，后半通道用 BN      ↓Global Average Pooling      ↓FC → 输出 256 或 128 维特征向量（你改为 128 维以节省存储）
2.2 数据集改造：PRCC 上半身专用版本
PRCC 原始数据： - 221 人，3 个摄像头（A、B、C） - A 摄像头：衣服 1；B、C 摄像头：衣服 2 - 室内实验室场景，全身像，分辨率较高
上半身裁剪策略：
原始图像 + 检测框 [x, y, w, h]      ↓人体姿态估计（轻量模型，如 OpenPose 或直接按框比例裁剪）      ↓确定上半身范围：从头顶到髋关节（约身高的 55-60%）      ↓裁剪区域：[x, y, w, int(h×0.6)]      ↓Resize 到 128×64（保持宽高比，短边填充或中心裁剪）      ↓数据增强：  - 随机水平翻转（上半身左右对称性弱，谨慎使用）  - 随机裁剪（模拟检测框抖动）  - 颜色抖动（亮度、对比度，模拟光照变化）  - 高斯模糊（模拟焦距变化）  - 注意：不做随机擦除（上半身区域小，擦除可能删掉关键身份区域）
训练/验证划分： - 标准 PRCC 协议：A 摄像头训练，B/C 测试 - 你的改造：按人划分，确保训练集和测试集人完全不重叠 - 子集划分：同衣服测试（A→A）、跨衣服测试（A→B/C）
2.3 微调策略：分层冻结与渐进解冻
核心思想：OSNet-IBN 在 ImageNet 上预训练了底层视觉特征（边缘、纹理、颜色），在 Market-1501/Duke 上预训练了中层行人特征（部件、姿态）。这些对你的上半身场景部分有用。顶层需要重新学习”上半身身份特征”。
阶段 A（Epoch 1-10）：硬冻结  - 冻结：Stem + OSNet Block 1-2（底层边缘/纹理）  - 训练：OSNet Block 3-4 + 聚合层 + FC（中上层 + 分类头）  - 学习率：1e-3（较高，快速适配新域）  - 目标：让模型快速学会"上半身有什么特征"阶段 B（Epoch 11-20）：软微调  - 解冻：OSNet Block 2（渐进）  - 训练：Block 2-4 + 顶层  - 学习率：1e-4（降低，精细调整）  - 目标：让中层也适应上半身尺度分布阶段 C（Epoch 21-30）：全微调（可选）  - 解冻：全部层  - 学习率：1e-5（很低，防止破坏预训练知识）  - 目标：全局优化，但风险是过拟合小数据集  - 早停：验证集跨衣服 Rank-1 连续 3  epoch 不提升则停止
关键决策：PRCC 只有 221 人，数据量小。全微调容易过拟合。建议阶段 A 必须做，阶段 B 视验证指标决定，阶段 C 通常跳过。
2.4 损失设计：身份 + 度量 + 换装一致性
多任务联合损失：
L_total = L_id + λ_triplet × L_triplet + λ_center × L_centerL_id：Softmax 交叉熵  - 输入：128 维特征 → FC(128→221) → Softmax  - 目标：正确分类身份  - 作用：提供强身份监督L_triplet：Batch Hard Triplet Loss  - 采样：每个 batch 选 P 人，每人 K 张（PK 采样）  - 关键改造：强制跨衣服采样    - 正样本：同一人、不同衣服    - 负样本：不同人、同一件衣服 或 不同人、不同衣服  - 目标：同一人跨衣服距离 < 不同人同衣服距离  - 作用：核心换装能力来源L_center：Center Loss（可选）  - 为每个身份维护一个特征中心  - 约束样本特征靠近中心，增强类内紧凑性  - 对跨衣服场景尤为重要：迫使同一人的不同衣服特征聚集
换装一致性约束（核心创新点）：
对同一人、不同衣服的两张图：  x_i（衣服 1）→ f_i（128 维特征）  x_j（衣服 2）→ f_j（128 维特征）约束：cosine_similarity(f_i, f_j) > α（如 0.6）实现方式：  - 在 Triplet 采样中，强制包含跨衣服正样本对  - 或增加显式损失：L_consistency = max(0, α - cos(f_i, f_j))
2.5 验证指标与导出
验证重点： - 同衣服 Rank-1/mAP：验证基础识别能力（应 > 90%） - 跨衣服 Rank-1/mAP：核心指标（目标 > 50%） - 跨衣服 mAP 比 Rank-1 更重要：反映检索列表整体质量
ONNX 导出：
训练完成 → 加载最佳验证 epoch 权重      ↓模型设为 eval 模式（BN 用 running statistics，非 batch statistics）      ↓构造 dummy input：torch.randn(1, 3, 128, 64)      ↓torch.onnx.export(model, dummy_input, "osnet_upperbody.onnx",                  opset_version=11,                  input_names=["input"],                  output_names=["feature"],                  dynamic_axes={"input": {0: "batch_size"},                               "feature": {0: "batch_size"}})      ↓验证 ONNX 输出与 PyTorch 输出一致性（误差 < 1e-5）
三、阶段二：模型压缩与量化
3.1 结构精简：面向 128×64 输入优化
原始 OSNet-IBN-x0.25 假设 256×128 输入。改为 128×64 后，特征图尺寸减半，部分层可精简：
输入降维：256×128 → 128×64      ↓Stem 层：保持，但第一个 Conv stride 可从 2 改为 1  - 原设计：大输入需要快速下采样  - 你的场景：输入已小，避免过度下采样导致信息丢失      ↓Block 1-2：通道数保持 x0.25（已很精简）      ↓Block 3-4：可考虑进一步压缩至 x0.125  - 但需谨慎：上层特征对身份判别更关键      ↓聚合层：多尺度权重维度相应调整      ↓输出：128 维（而非 256 或 512）  - 理由：你的 Gallery 只有 3-10 人，128 维足够区分  - 存储节省：每条模板 128×4 = 512 字节（FP32）或 128 字节（INT8）
3.2 量化策略：INT8 后训练量化（PTQ）
ESP32-P4 无 NPU，纯 CPU 推理。INT8 比 FP32 快 2-4 倍，且省 75% 内存。
PTQ 流程：
1. 准备校准集：从 PRCC 验证集抽 100-500 张代表性上半身图      ↓2. 插入 Quantize-Dequantize 节点：   - 输入：FP32 → INT8（scale/zero_point  per-tensor 或 per-channel）   - 权重：INT8（per-channel 通常更好）   - 激活：INT8（per-tensor 通常足够）      ↓3. 校准：前向传播校准集，统计激活分布，确定 optimal scale      ↓4. 验证：INT8 模型 vs FP32 模型，特征余弦相似度 > 0.99   - 若差距大 → 改用 QAT 或更多校准数据      ↓5. 导出：TFLite FlatBuffer 格式
量化感知训练（QAT）备选： - 若 PTQ 后精度损失大（跨衣服 Rank-1 下降 > 5%） - 在训练最后 5-10 epoch 模拟 INT8 前向，FP32 反向 - 计算开销大，但精度恢复更好
3.3 格式转换：TFLite Micro 适配
ESP32-P4 的 RISC-V CPU 可选支持： - 标准 TFLite Micro：纯 C++，无 SIMD 优化，最慢但最通用 - ESP-DL：乐鑫官方优化库，支持部分算子加速 - 自定义：手写关键卷积循环，利用 RISC-V P 扩展（向量指令）
决策：
优先 TFLite Micro：  - 生态成熟，OSNet 的算子（Conv, BN, ReLU, AvgPool, FC）全支持  - 若推理太慢（> 500ms），再考虑 ESP-DL 或自定义  TFLite Micro 集成：  - 将 .tflite 模型文件烧录到 SPIFFS  - 运行时加载到 RAM（模型大小 < 500KB）  - 解释器初始化：分配 tensor 内存（输入 + 中间特征图 + 输出）  - 推理：Invoke() 一次，输出 128 维特征向量
四、阶段三：ESP32-P4 嵌入式部署
4.1 推理引擎选择与内存架构
ESP32-P4 资源评估： - CPU：双核 RISC-V，最高 400MHz（实际可用约 300-350MHz） - RAM：内部 SRAM 约 512KB，外部 PSRAM 可选 8-16MB - Flash：外部 SPI Flash，通常 8-16MB - 无 NPU，无硬件浮点加速（需确认具体型号）
内存布局设计：
SPI Flash（只读，大容量）：  ├── osnet_model.tflite（~300-500KB，INT8）  ├── reid_gallery.json（~20KB，3人×12模板）  └── 其他资源（TTS 语音包、UI 素材等）PSRAM（若可用，读写，较慢）：  └── 模型运行时加载备选（若内部 RAM 不够）内部 SRAM（_fast，稀缺）：  ├── TFLite 解释器工作区（~100KB）  │   ├── 输入 tensor：128×64×3 = 24KB（INT8）  │   ├── 中间特征图（最大层）：如 32×16×64 = 32KB  │   └── 输出 tensor：128 = 0.1KB  ├── 当前帧 ROI 缓存（~30KB）  └── 其他应用状态（UI、语音等）
关键约束：内部 SRAM 可能只有 512KB，需与其他任务（LVGL、音频缓冲）共享。若紧张，考虑： - 模型权重从 Flash 流式加载（分页），而非全加载到 RAM - 或启用 PSRAM，但接受 10-20% 速度损失
4.2 预处理流水线（设备端）
human_detect 输出行人 bbox（在 1024×600 帧中）      ↓ROI 提取：  - 原图是 RGB565，直接操作  - 根据 bbox 裁剪行人区域  - 上半身适配：取 bbox 上半 60%（或全身 resize 后上半部分）      ↓Resize 到 128×64：  - 双线性插值（质量 vs 速度权衡）  - 或最近邻（最快，质量稍差但 Re-ID 对精细纹理不敏感）      ↓颜色空间转换：  - RGB565 → RGB888（8 位 per channel）  - 或直接用 RGB565 的 R/G/B 分量近似（省转换开销）      ↓归一化：  - 减去 ImageNet 均值：[0.485, 0.456, 0.406] × 255  - 除以标准差：[0.229, 0.224, 0.225] × 255  - 量化到 INT8：((fp32_value / 255) - mean) / std × 255 × scale      ↓输入 TFLite 解释器
耗时预估： - ROI 裁剪 + Resize：~10ms（纯 C，无优化） - 归一化 + 量化：~5ms - TFLite 推理：~100-300ms（取决于模型大小和优化程度） - 总计：120-320ms（可接受，非实时每帧，只在状态变化时触发）
4.3 后处理与特征匹配
TFLite 输出：128 维 INT8 或 FP32 特征向量      ↓L2 归一化（关键！）：  - 计算向量模长：sqrt(sum(x_i^2))  - 每个元素除以模长  - 结果：单位向量，模长 = 1  - 作用：消除光照、尺度变化影响，使余弦相似度 = 点积      ↓Gallery 加载（从 SPIFFS 到 RAM）：  - 解析 reid_gallery.json  - 所有模板特征已预归一化（注册时完成）      ↓视角过滤（可选）：  - 若启用 Micro-ViewNet：先估计当前视角  - 只搜索同视角 + 相邻视角桶（减少计算量，提升精度）  - 若视角估计置信度低：搜索全部模板      ↓批量余弦相似度计算：  - score_i = dot(query_feature, gallery_template_i)  - 范围 [-1, 1]，1 表示完全相同      ↓决策：  - max_score > SELF_THRESHOLD（如 0.75）：匹配成功，返回身份  - max_score < STRANGER_THRESHOLD（如 0.45）：陌生人  - 中间：UNKNOWN，可触发 VLM 确认或要求重新注册
阈值调优策略： - 注册阶段：记录同一人不同模板间的相似度分布，确定 SELF_THRESHOLD 下限 - 测试阶段：记录不同人模板间的相似度分布，确定 STRANGER_THRESHOLD 上限 - 动态调整：根据场景反馈（用户纠正）在线微调
五、阶段四：系统集成与业务闭环
5.1 Gallery 存储结构
SPIFFS 文件：reid_gallery_v2.bin（二进制，紧凑格式）文件头（32 字节）：  - magic: "REID" (4 bytes)  - version: 2 (2 bytes)  - num_identities: 3 (2 bytes)  - templates_per_identity: 12 (2 bytes)  - feature_dim: 128 (2 bytes)  - quantization_type: INT8 (1 byte) 或 FP32 (1 byte)  - reserved: 17 bytes模板记录（每条 140 bytes @ INT8）：  - identity_id: uint8 (1 byte)  - name: char[16] (16 bytes, UTF-8, 如 "小明")  - view: uint8 (1 byte, 0=front, 1=side, 2=back, 3=unknown)  - feature[128]: int8 (128 bytes, 已 L2 归一化)  - enrolled_at: int64 (8 bytes, Unix timestamp)  - hit_count: uint32 (4 bytes, 命中次数，用于模板更新权重)  - confidence: uint8 (1 byte, 注册时的平均置信度)  - reserved: 3 bytes总容量：3 × 12 × 140 + 32 = 5072 bytes ≈ 5KB
5.2 注册流程：语音触发 + 多帧采集 + 质量筛选
触发：用户说"记住我，我叫小明"      ↓状态机：IDLE → ENROLLING      ↓TTS："请坐在屏幕前，保持自然姿势"      ↓连续采集 5 秒，帧率 5fps → 25 帧      ↓每帧处理：  - human_detect 确认"在座"  - 提取 ROI，OSNet 推理得 128 维特征  - 估计视角（若启用 ViewNet，或基于宽高比规则）  - 质量评分：    * 检测置信度 > 0.8    * 人脸区域不过度遮挡（可选，用简单规则）    * 特征向量模长正常（非全黑/全白输入）      ↓按视角分桶，每桶保留最佳 2 帧：  - 最佳标准：检测置信度 × 特征范数（反映图像清晰度）      ↓写入 Gallery：  - 检查同名身份是否存在 → 更新或新增  - 若超出 3 人限制：提示删除旧身份  - SPIFFS 写入，双备份（原文件 + .bak）      ↓TTS："已经记住你了，小明"状态机：ENROLLING → IDLE
5.3 识别流程：与现有 human_detect 对接
on_human_present() 触发：  - 确认连续 3 帧检测稳定（防抖）      ↓调用 reid_identify(current_frame)：  - 提取 ROI（与注册时相同的裁剪逻辑）  - OSNet 推理 → 128 维特征  - L2 归一化      ↓Gallery 匹配：  - 加载所有模板到 RAM（5KB，可接受）  - 批量点积计算  - 取 Top-3 候选      ↓决策与状态更新：  - 若 max_score > 0.75：    * identity_name = "小明"    * identity_matched = true    * 更新该模板 hit_count++    * 触发移动平均更新（见 5.5）    * UI 显示"小明 在座"    * 若状态从离座恢复：TTS"欢迎回来，小明"    - 若 max_score < 0.45：    * identity_name = "陌生人"    * identity_matched = false    * 若当前处于 MONITORING 模式：触发报警流程    * UI 显示"陌生人"    - 若 0.45 ≤ max_score ≤ 0.75：    * identity_name = "未知"    * identity_matched = false    * UI 显示"检测中..."    * 可选：连续 3 次 UNKNOWN 后触发 VLM 确认      ↓返回 identity_id 给状态机
5.4 监控闭环：白名单过滤
monitor_start()：  - 保存当前参考帧（JPEG，用于 VLM 对比）  - 记录当前在场身份为白名单（如"小明"）  - TTS："已开始监测，有陌生人进入我会提醒你"      ↓on_pir_motion_detected()：  - 唤醒摄像头，捕获当前帧  - reid_identify(current_frame)      ↓结果判定：  - 白名单身份（score > 0.75）：忽略，不打扰  - 陌生人（score < 0.45）：    * 调用 VLM 对比参考帧与当前帧    * VLM 确认"有显著变化" → TTS"检测到陌生人进入学习区域"    * 更新参考帧  - UNKNOWN：    * VLM 确认 → TTS"检测到未知人员，请确认"      ↓monitor_stop()：  - 清除参考帧与白名单  - TTS："已结束监测"
5.5 在线更新：模板漂移与老化
高置信度命中时（score > 0.85）：  - 当前特征与匹配模板做移动平均：    template_feature_new = 0.9 × template_feature_old + 0.1 × query_feature  - 重新 L2 归一化  - 更新 hit_count  - 作用：适应缓慢的外观变化（如发型微调、眼镜摘戴）定期维护（每天凌晨或空闲时）：  - 扫描 Gallery，检查模板年龄  - 若某模板 30 天未命中且 hit_count < 5：标记为"冷模板"  - 注册新模板时优先替换冷模板异常检测：  - 若同一身份连续 10 次匹配分数下降（从 0.9 → 0.6）：    * 可能外观剧变（如剪发、换眼镜）    * 提示用户："检测到小明变化较大，建议重新注册"
