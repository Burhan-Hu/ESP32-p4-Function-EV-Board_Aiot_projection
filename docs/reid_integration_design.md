# 结合跨外观行人重识别（Re-ID）的功能重构方案

> 基于刘蒙蒙《基于深度学习的跨外观行人重识别关键技术研究》（112019017）与当前 ESP32-P4 AIoT 智能体项目现状整理。

---

## 一、博士论文核心技术讲解

### 1.1 研究问题

论文聚焦 **跨外观行人重识别（Cross-appearance Person Re-ID）**：现实监控中，同一个人的衣着、配饰、姿态、拍摄角度、光照会随时间发生剧烈变化，传统 Re-ID 假设“同一人外观不变”，导致识别失效。论文从 **特征表达**、**数据增强**、**视角建模** 三个维度降低模型对外观（衣服颜色/纹理）的依赖，转而学习体型、轮廓、姿态、步态等身份不变特征。

### 1.2 主要技术模块

| 模块 | 章节 | 核心思想 | 可借鉴点 |
|------|------|----------|----------|
| **NKUP+ 数据集** | 第二章 | 长时间跨度、多摄像头、跨季节采集，标注身份/外观/视角标签 | 本项目做“人在座/离座”长期跟踪时，也需要积累同一人多外观、多视角样本 |
| **M2Net** | 第三章 | 输入 RGB + 轮廓草图（RCF）+ 人体语义分割（PSPNet）三种模态；提出 **RAS 随机外观采样** 与 **CA 跨外观损失**，让模型在同身份不同外观的样本上拉近距离 | 端侧可退化为“颜色+边缘/轮廓”多模态手工/轻量特征，提升换衣服后的识别稳定性 |
| **CAG** | 第四章 | 用 GAN 解耦行人的 **外观编码** 与 **身份/体型编码**，生成“同一人、不同衣服”的合成样本做数据增强 | 可在 PC/手机端离线生成合成样本，扩充设备上注册的模板库 |
| **MVMA** | 第五章 | 显式建模前/侧/后视角，用 **全局分支 + 局部分支** 提取特征，并设计 **跨视角损失（CV Loss）** 与视角正则化 | 端侧可按视角分桶匹配（只比同/近视角模板），降低误匹配 |

### 1.3 论文对本项目的最大启示

1. **不要只依赖 RGB 外观**。衣服一换，纯颜色/纹理特征会失效。
2. **把身份特征和外观特征解耦**。身份特征应锁定在体型、轮廓、姿态、头身比等稳定属性。
3. **视角是独立扰动源**。正面与侧面的轮廓差异巨大，匹配时应按视角分组。
4. **数据增强可以离线做**。用生成模型在注册阶段扩充“同一人、多外观”模板，比在线训练 GAN 更现实。

## 二、项目现状与 Re-ID 结合点分析

### 2.1 已具备的能力

- **视觉**：SC2336 → ISP → RGB565 1024×600，已有 JPEG 编码、`camera_capture_jpeg()`。
- **端侧人体检测**：`human_detect` 组件运行 ESP-WHO `PedestrianDetect`，维护 `在座/离座` 状态、超时、离开提醒。
- **交互**：语音唤醒 → ASR → VLM → 豆包 TTS；支持“开始监测/结束监测”。
- **监控**：PIR 触发后调用 `cloud_vlm_ask_with_reference()` 对比前后帧，检测画面变化。
- **UI**：屏幕显示 seat_state、screen_state、risk_level、学习时长等。

### 2.2 当前短板

| 短板 | 影响 |
|------|------|
| 人体检测只有“有人/无人”二值 | 无法区分“是学生本人回来”还是“别人经过”；无法做个性化伴学 |
| 没有身份记忆 | 离开再回来会重置状态，无法累计“今日专注时长” |
| 监控模式只比对画面内容 | 无法判断“是不是同一个人”，误报率高 |
| 外观/视角变化无建模 | 学生换件衣服、侧身/背面坐下，系统就“不认识” |
| 云端 VLM 调用贵且慢 | 不适合每帧做人脸/人体比对 |

### 2.3 Re-ID 的切入点

把设备从“检测到有人”升级为 **“识别到是谁，并据此提供连续、个性化的服务”**。具体切入点：

1. **座位身份感知**：在座/离座状态增加身份维度（本人/他人/未知）。
2. **跨会话学习档案**：同一人离开后返回，恢复个人学习计时、未完成作业提醒。
3. **智能监控白名单**：监测模式下，只对外来者（非注册用户）产生语音提醒。
4. **换衣服/换姿势鲁棒**：借鉴 M2Net 与 MVMA，用轮廓+体型+视角分组，降低外观依赖。

## 三、面向需求的 Re-ID 功能重构方案

### 3.1 设计目标

- **边缘优先**：Re-ID 推理必须在 ESP32-P4 本地完成，不能每帧上云。
- **轻量化**：模型/特征存储占用 < 50 KB，单次推理 < 100 ms。
- **可落地**：不追求论文级精度，追求“能用、稳、可演示”。
- **与现有链路融合**：复用现有的人体检测、PIR、VLM、TTS、状态机、UI。

### 3.2 新增/增强功能清单

| 功能 | 说明 | 改动模块 |
|------|------|----------|
| **身份注册** | 语音“记住我”后，连续抓拍 3~5 张不同角度的人像，提取特征写入模板库 | Re-ID 组件 + 唤醒交互 |
| **入座识别** | `human_present` 触发后，先运行 Re-ID；若匹配到注册身份，显示“欢迎回来，XXX”并恢复个人状态 | human_detect、状态机、UI |
| **离座身份保持** | 离开后模板不删，保留个人学习时长；超时未归可触发“学习暂停” | 状态机 |
| **跨外观匹配** | 换衣服/侧身/背面时，通过轮廓+体型+视角分组仍能识别本人 | Re-ID 特征设计 |
| **陌生人提醒** | 监测模式下，若当前人物与注册身份不匹配，播报“检测到陌生人进入” | PIR/监控链路 |
| **多用户切换** | 支持 2~3 个家庭成员注册，自动切换对应的学习档案 | UI + 状态机 |

### 3.3 端侧 Re-ID Lite 架构

```text
RGB565 帧
   │
   ▼
PedestrianDetect ──► 行人 bbox（score + 框）
   │
   ▼
Crop & Resize (128×64 或 128×256)
   │
   ├──► 颜色分支：HSV/YUV 直方图 ──┐
   ├──► 轮廓分支：边缘/剪影 + 宽高比 ──┼──► 拼接特征向量 ──► 视角分组
   └──► 体型分支：头肩/躯干比例（可选）──┘        │
                                                ▼
                                       与 Gallery 同视角模板比距离
                                                │
                                                ▼
                                       输出：identity_id + confidence
```

**关键点**：
- **多模态特征** 对应论文 M2Net 思想，但用可实时计算的手工/轻量特征替代 RCF+PSPNet。
- **视角估计** 用 bbox 宽高比 + 行人中心位置粗分为 front/side/back，对应 MVMA 的按视角匹配。
- **度量学习** 用余弦距离或加权欧氏距离，设定阈值判定“本人/他人/未知”。

### 3.4 Gallery（模板库）设计

- 存储位置：SPIFFS 或 NVS（每人特征 < 5 KB）。
- 结构：
  ```c
  typedef struct {
      uint8_t  id;
      char     name[16];
      uint8_t  view;            // front/side/back
      float    feature[DIM];    // 颜色+轮廓+体型拼接
      int64_t  enrolled_at;
  } reid_template_t;
  ```
- 离线增强：在 PC/手机上用论文 CAG 思想生成“同一人、不同衣服/背景”的合成图，提取特征后写入设备，提高跨外观鲁棒性。

### 3.5 与现有状态机/云 API/UI 的融合

| 现有模块 | 融合方式 |
|----------|----------|
| `human_detect` | 检测到有人在座后，回调 `on_human_present()` 中调用 `reid_identify()`，更新 `s_ui_model.identity_id` |
| `monitor_start/stop` | 启动监测时保存当前用户身份；PIR 触发后，若 Re-ID 结果 ≠ 注册身份，才调用 VLM 确认并播报 |
| `cloud_vlm_ask` | 仅作为 **语义 Re-ID fallback**：当本地置信度处于阈值边缘时，上传两张裁剪图问“是否为同一人” |
| `cloud_tts_speak` | 识别结果驱动个性化语音：本人回来 → “欢迎继续学习”；陌生人 → “检测到陌生人” |
| UI | `seat_state` 由“在座/离座”扩展为“XXX 在座 / 陌生人 / 检测中”；增加“今日专注时长”卡片 |
| 状态机 | 新增 `STATE_IDENTIFYING`、`STATE_ENROLLING`；在座超时未归转入 `STATE_PAUSE` |

### 3.6 推荐落地阶段

| 阶段 | 任务 | 验收标准 |
|------|------|----------|
| **Phase 1** | 实现 Re-ID 特征提取（颜色+轮廓）与 Gallery 注册/匹配 | 同一人前后 5 分钟内入座，识别准确率 > 80% |
| **Phase 2** | 加入视角分组（前/侧/后）与陌生人提醒 | 侧身/背面仍能被识别；非注册用户触发提醒 |
| **Phase 3** | 结合 PIR/监控模式，实现“白名单式”安防 | 只有陌生人进入监测区域才上云确认并播报 |
| **Phase 4** | 离线数据增强：用 PC 生成合成样本扩充 Gallery | 换衣服场景识别率提升 > 15% |

## 四、可行性与风险

### 4.1 可行性

- ESP32-P4 有 PIE AI 加速和充足 PSRAM，运行轻量特征提取完全可行。
- 当前 `human_detect` 已经提供稳定的人体存在信号，Re-ID 只需在“状态变化”时触发，不需要每帧跑。
- Gallery 很小，可持久化到 SPIFFS；注册流程可通过语音命令完成，交互自然。

### 4.2 风险与应对

| 风险 | 应对 |
|------|------|
| Re-ID 精度不足 | 以“辅助身份感知”为定位，不用于关键安全决策；低置信度走“未知/请确认”流程 |
| 视角估计不准 | 允许跨相邻视角桶匹配（front↔side），back 单独处理 |
| 存储空间不够 | 只存特征向量，不存原图；限制注册人数 ≤ 3 |
| 计算延迟 | 仅在 `human_present` 变化瞬间跑一次 Re-ID，平时不占用 CPU |
| 与 VLM 冲突 | Re-ID 结果用于本地决策，VLM 仅做二次确认，降低云端调用次数 |

---

## 五、结论

论文给出的跨外观 Re-ID 方法论（多模态特征、外观-身份解耦、视角感知、数据增强）可以直接映射到本项目的 **“轻量身份感知层”** 设计中。我们不必在 ESP32-P4 上复现 DenseNet121 + GAN，而是吸收其核心思想，用 **边缘轻量特征 + 离线生成增强 + 视角分桶匹配** 的方式，为 AIoT 伴学终端增加“认人”能力，从而实现：

- 个性化的学习陪伴与连续计时；
- 监测模式下的陌生人过滤与精准提醒；
- 对外观变化（换衣服、侧身、背影）更强的鲁棒性。

这与乐鑫赛道要求的 **“边缘 AI 计算、传感器融合、多模态交互、创新场景”** 高度契合，是项目竞争力的重要加分项。
